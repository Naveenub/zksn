## Verification

Anyone can verify the ceremony chain:

```bash
npm install snarkjs circom2 circomlib

# Recompile circuit (confirms r1cs matches)
./node_modules/.bin/circom2 circuits/MembershipVote.circom \
    --r1cs --wasm --sym --output build/

# Verify zkey chain against r1cs + ptau
npx snarkjs zkey verify \
    build/MembershipVote.r1cs \
    ceremony/pot15_final.ptau \
    ceremony/zkey_final.zkey

# Verify a proof
npx snarkjs groth16 verify \
    ceremony/verification_key.json \
    ceremony/public.json \
    ceremony/proof.json
```

Expected output: `ZKey Ok!` / `OK!`

## Upgrade path — mainnet

Replace pot15 with Hermez pot28 for maximum phase 1 security:

```bash
# Download Hermez pot28 (https://github.com/iden3/snarkjs#7-prepare-phase-2)
wget https://hermez.s3-eu-west-1.amazonaws.com/powersoftau28_hez_final.ptau \
     -O ceremony/pot28_final.ptau

# Re-run phase 2 against pot28 — scripts/ceremony.sh handles the full flow
bash scripts/ceremony.sh init   # uses PTAU=ceremony/pot28_final.ptau
```

The circuit, witness, and Solidity interface do not change — only the ptau
file and resulting zkey. pot28 has 1000+ contributors from the Hermez ceremony,
giving the strongest available phase 1 trust assumption.
