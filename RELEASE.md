# ZKSN v0.10.0-alpha — pot28 ceremony scaffolding, encode_proof.js, ceremony.sh hardened
**⚠️ Pre-alpha research prototype. Not production ready.**

## What's in this release

### Phase 1 trust gap documented and scaffolded

The gap from v0.9.0 was: pot15 was generated by 3 project contributors.
If any project contributor retained their phase 1 toxic waste, they could
forge arbitrary membership proofs. The circuit's zero-knowledge property
depends on phase 1 soundness.

This release provides everything needed to close that gap with Hermez pot28.

### `scripts/download_ptau.sh` — NEW

Downloads and verifies Hermez pot28 with SHA256 integrity check.

```bash
bash scripts/download_ptau.sh
# Downloads ceremony/pot28_final.ptau (~3.2 GB)
# Verifies: SHA256 55c77ce8562366c91e7cda394cf7b7c15a06c12d8c905e8b36ba9cf5e13eb37d
# Then: bash scripts/ceremony.sh init
```

The script tries both Hermez S3 and Google Cloud mirrors with retry logic.
If both fail it prints the URL and exits cleanly with instructions.

### `scripts/ceremony.sh` — defaults to pot28

```diff
- PTAU="ceremony/pot15_final.ptau"
+ PTAU="ceremony/pot28_final.ptau"   # default: Hermez mainnet
```

Dev fallback (no download):
```bash
PTAU=ceremony/pot15_final.ptau bash scripts/ceremony.sh init
```

The `init` command now fails with a clear message if pot28 is missing,
rather than silently using a wrong or absent file. Post-ceremony checklist
is printed after `finalize` — exact steps to update `Groth16Verifier.sol`,
regenerate the proof, and update test constants.

### `scripts/encode_proof.js` — NEW

Encodes a `snarkjs` proof.json to the correct EIP-197 Solidity hex literal.

```bash
node scripts/encode_proof.js ceremony/proof.json
```

Output is ready to paste directly into `ZKSNGovernance.t.sol`:

```solidity
bytes constant REAL_PROOF =
    hex"2f5d04b4c973e94a21808f2c9ccd71ab8b0eb4ff3de96b7ea65103944eb297d7"
    hex"16e4aaecf73c6a01d6ffc8687a0d7df4bd1f9e7303338a3f3b00dcc91cf6281b"
    ...
```

This script is the canonical fix for the Solidity `Expected even number of
hex-nibbles` error — it validates that every chunk is exactly 64 hex chars
(32 bytes) before printing, so malformed output is caught immediately rather
than at `forge build`.

### `ceremony/ATTESTATION.md` — pot15 vs pot28 trust comparison

Documents both ceremonies side by side with an explicit trust comparison
table. The pot28 section includes SHA256, URL, download command, and the
exact post-ceremony steps required to update all Solidity files.

## What changes in Groth16Verifier.sol after pot28 ceremony

Only delta and IC change (phase 2 re-randomizes them). alpha, beta, gamma
are circuit-structure constants — they are identical across any phase 2
run against the same r1cs and phase 1:

```
alpha*, beta*, gamma*, IC0..IC4 — UNCHANGED (circuit structure)
delta*                          — NEW (phase 2 re-randomized)
```

The assembly verification body in `_verifyProof` does not change.

## Full pot28 ceremony runbook

```bash
# 1. Download (one-time, ~3.2 GB)
bash scripts/download_ptau.sh

# 2. Phase 2 init (uses pot28 by default)
bash scripts/ceremony.sh init

# 3. Three independent contributors (each on a different machine)
bash scripts/ceremony.sh contribute 1 ceremony/zkey_0000.zkey ceremony/zkey_0001.zkey
bash scripts/ceremony.sh contribute 2 ceremony/zkey_0001.zkey ceremony/zkey_0002.zkey
bash scripts/ceremony.sh contribute 3 ceremony/zkey_0002.zkey ceremony/zkey_0003.zkey

# 4. Finalize + export VK + Solidity verifier
bash scripts/ceremony.sh finalize

# 5. Regenerate proof
node scripts/tree.js add 12345678901234567890
node scripts/tree.js input 12345678901234567890 999888777666555 1 > ceremony/input.json
node build/MembershipVote_js/generate_witness.js \
     build/MembershipVote_js/MembershipVote.wasm \
     ceremony/input.json ceremony/witness.wtns
npx snarkjs groth16 prove ceremony/zkey_final.zkey \
     ceremony/witness.wtns ceremony/proof.json ceremony/public.json

# 6. Encode proof for Solidity
node scripts/encode_proof.js ceremony/proof.json

# 7. Update governance/contracts/Groth16Verifier.sol (copy delta* and IC*)
# 8. Update governance/test/ZKSNGovernance.t.sol (REAL_PROOF, REAL_MEMBERSHIP_ROOT)
# 9. forge test
# 10. Update ceremony/ATTESTATION.md
# 11. Commit: 'feat/ceremony-mainnet: pot28 N-party trusted setup'
```
